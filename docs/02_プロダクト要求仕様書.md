# 公文書庫 プロダクト要求仕様書（PRD)

- 作成日: 2026-07-06
- バージョン: 0.1（ドラフト）
- 関連文書: [01_企画書.md](01_企画書.md)

## 1. 背景と目的

e-Govからダウンロードした電子公文書（XML＋XSL＋添付）は、①ブラウザのXSLT廃止（Chrome 158 / 2026-11-17）により従来の閲覧方法が失われ、②e-Gov側の取得期限（発行後3か月）により手元のZIPが唯一の原本となる。本プロダクトは、公文書を**恒久的に保全し、整理・検索・閲覧できるローカルWebアプリ**を提供する。

**目的（優先順）:**

1. 公文書をブラウザ環境に依存せず、いつでも正しく表示できること
2. 原本を無加工で保全し、原本性（改ざんの有無）を確認できること
3. 到達番号の羅列から解放され、機関・日付・件名で文書を見つけられること

## 2. 対象ユーザーと利用環境

| 項目 | 内容 |
|---|---|
| ユーザー | 単一ユーザー（開発者本人）。認証なし |
| 実行環境 | macOSローカル。サーバーは `localhost` のみにバインドし、外部公開しない |
| ブラウザ | Chrome / Safari / Firefox の最新版。**ブラウザのXSLT・プラグイン等に依存しない** |
| ネットワーク | 完全オフラインで動作すること（外部送信なし） |
| データ保管 | ローカルディスク上の単一の書庫ディレクトリ |

## 3. スコープ

**スコープ内:**

- e-Gov公文書ZIP・フォルダの取り込みとカタログ化
- 一覧・絞り込み・検索
- 鑑XML／様式XMLのサーバー側XSLT変換による閲覧、PDF・CSVの閲覧
- 原本の無加工保全と、署名ダイジェストによる改ざん検知

**スコープ外（当面）:**

- 複数ユーザー、認証、クラウドホスティング
- e-Gov APIによる公文書の自動取得
- GPKI証明書チェーンの完全な署名検証（Phase 3で再検討）
- 公文書の編集・加工（本プロダクトは読み取り専用アーカイブ）

## 4. 用語

| 用語 | 定義 |
|---|---|
| 公文書 | e-Govから交付される電子文書一式。ZIP形式で配布される |
| 到達番号 | 申請ごとに振られる一意の番号（例: `209906019912345678`）。文書セットの主キー |
| 鑑XML（かがみ） | 文書セットの表紙にあたるXML。メタデータとXMLDSig署名を含む |
| 様式XML | 通知書本体などのXML。専用XSLで表示する |
| 対のXSL | XML内 `<?xml-stylesheet?>` が参照するXSLTスタイルシート |
| 書庫 | 本システムが管理するローカルのルートディレクトリ |
| 文書セット | 到達番号1件に対応するファイル一式（鑑・XSL・添付・元ZIP） |

## 5. 機能要求

優先度: **P0** = MVP必須 / **P1** = MVP後すみやかに / **P2** = 将来

### FR-01 取り込み（P0）

- ZIPファイルをWeb UIからアップロード（ドラッグ&ドロップ含む）して取り込める
- 解凍済みフォルダのパス指定でも取り込める
- 取り込み時の処理: 解凍 → 鑑XMLの特定 → メタデータ抽出 → 書庫へ複製（`書庫/到達番号/`）→ 元ZIPも `書庫/到達番号/` 配下に保管 → カタログ登録
- 同一到達番号の再取り込みは既定でスキップし、その旨を表示する（上書きはしない）
- 取り込み元ファイルは削除しない（ユーザーの手元のファイルに副作用を与えない）
- 鑑XMLが特定できない・解凍に失敗するなどの場合は、理由を明示してエラーにする（部分的に壊れた状態で登録しない）

### FR-02 メタデータ抽出（P0)

- 鑑XMLから以下を抽出する: 到達番号、受付日／発行日、発行機関、件名、添付ファイル一覧
- 和暦（例: `令和 8年 6月27日`）は西暦に正規化して保持する（表示は和暦併記可）
- 既知パターン（`TOTATUNO` 系＝厚労省審査結果、`DOCNO` 系＝年金機構 kagami 等）を順に試行し、パターンは追加式で拡張できる構造にする
- 全パターンに合致しない場合もエラーにせず、ファイル名・更新日時からのフォールバック値で登録し、「形式未対応」フラグを付ける

### FR-03 一覧・絞り込み（P0）

- 全文書セットを一覧表示する。表示項目: 受付日、発行機関、件名、到達番号、添付の種類（PDF/XML/CSV）、取込日
- 受付日・取込日でソートできる
- 発行機関、年月で絞り込める
- 一覧は文書セット1,000件でも実用的な速度で表示できること（目安1秒以内）

### FR-04 閲覧（P0）

- 鑑XML・様式XMLを、対のXSLで**サーバー側でXSLT 1.0変換**し、HTMLとして表示する
  - XSLの `shift_jis` 出力指定に関わらず、ブラウザへは常にUTF-8で配信する
  - 変換後HTMLが旧式マークアップ（IE時代の属性）でも著しく崩れないよう、表示コンテナに軽いCSS正規化を当てる
  - e-Gov公式の表示手段（従来のブラウザ直接表示）と同等の内容が読めることを基準とする
- PDFはブラウザ内でインライン表示する
- CSVはShift_JISをUTF-8に変換し、テーブルとして表示する
- すべてのファイルは原本のままダウンロードもできる
- XSLが欠落・破損している場合は、XMLの内容を汎用整形（要素名と値のツリー表示）で表示するフォールバックを備える

### FR-05 全文検索（P1）

- 件名・発行機関・XML本文テキストを対象にフリーワード検索できる（SQLite FTS5想定）
- 日本語の部分一致で実用になること

### FR-06 改ざん検知（P1）

- 鑑XMLのXMLDSig `Reference` に含まれる添付ファイルのSHA-256ダイジェストと、実ファイルのハッシュを照合する
- 照合結果を文書セット詳細に「改ざんなし／不一致／検証不能」として表示する
- 証明書チェーンの検証（GPKI）は本要求に含まない（Phase 3）

### FR-07 過去分の一括取り込み（P1）

- 指定ディレクトリ（例: `~/Documents`）を走査し、e-Gov公文書と思われるZIP・フォルダを検出して一括取り込みできる
- 取り込み前に検出結果の一覧を提示し、ユーザーが確認してから実行する

### FR-08 監視フォルダ自動取り込み（P2）

- 指定フォルダ（例: `~/Downloads`）に公文書ZIPが置かれたら自動で取り込む

### FR-09 タグ・メモ（P2）

- 文書セットに任意のタグとメモを付けられる（原本には書き込まない）

## 6. 非機能要求

| ID | 分類 | 要求 |
|---|---|---|
| NFR-01 | 原本保全 | 取り込んだ原本ファイル（ZIP含む）は一切変更しない。閲覧用の変換はすべてオンザフライまたは別領域のキャッシュで行う |
| NFR-02 | 耐障害性 | DB（カタログ）が失われても、書庫ディレクトリの走査（再インデックス）だけで全機能が復元できる。真実の源は常にファイルシステム |
| NFR-03 | プライバシー | サーバーは `127.0.0.1` のみにバインド。外部ネットワークへの送信を一切行わない |
| NFR-04 | ブラウザ非依存 | XSLT・ActiveX・特定ブラウザAPIに依存しない。HTML/CSS/最小限のJSのみで動作 |
| NFR-05 | 性能 | 文書セット1,000件規模で、一覧表示1秒以内・XSLT変換2秒以内（目安） |
| NFR-06 | 文字コード | Shift_JIS / UTF-8 の混在を吸収し、UIでは文字化けを発生させない |
| NFR-07 | 起動の容易さ | 単一コマンド（`uv run` 等）で起動できる。ビルド工程を要しない |
| NFR-08 | 可搬性 | 書庫ディレクトリを丸ごとコピーすればバックアップ・移行が完了する |

## 7. データ要件

### 7-1. 書庫レイアウト

```
書庫ルート/（既定: ~/kobunshoko-archive/ ※初回起動時に設定可能）
├── catalog.db           … SQLiteカタログ（索引のみ。消えても再構築可能）
└── docs/
    └── {到達番号}/
        ├── original.zip … 取り込み時の元ZIP（ZIP取り込みの場合）
        └── （解凍された原本一式をそのまま配置）
```

### 7-2. カタログスキーマ（素案）

```
documents: 到達番号(PK), 受付日(西暦・和暦原文), 発行機関, 件名, 形式パターン名,
           取込日時, 署名検証結果, 形式未対応フラグ
files:     id(PK), 到達番号(FK), ファイル名, 役割(鑑/XSL/様式/PDF/CSV/その他),
           SHA-256, ダイジェスト照合結果
search:    FTS5仮想テーブル（件名, 発行機関, XML本文テキスト）
```

## 8. 画面要求

| 画面 | 主な要素 |
|---|---|
| 一覧（ホーム) | 文書セットの表（受付日・機関・件名・添付種別）、絞り込み（機関・年月）、検索ボックス（P1)、取り込みボタン兼ドロップゾーン |
| 文書セット詳細 | メタデータ、ファイル一覧（役割・サイズ・改ざん検知バッジ〈P1〉）、鑑のHTML表示、添付へのタブ/リンク |
| ファイル閲覧 | XML→変換済みHTML／PDFインライン／CSVテーブル。原本ダウンロードボタン |
| 取り込み結果 | 成功・スキップ（重複）・失敗の内訳と理由 |

## 9. 受け入れ基準（MVP = P0完了の定義）

1. 手元の2サンプル（労基署審査結果通知ZIP、年金機構通知フォルダ）を取り込むと、一覧に発行機関・受付日・件名が正しく表示される
2. 労基署サンプル: 鑑がe-Gov公式表示と同等のHTMLで表示され、控えPDFがブラウザ内で閲覧できる
3. 年金機構サンプル: 鑑と様式XML（納入告知額通知書）がそれぞれ対のXSLで表示され、CSV明細が文字化けなくテーブル表示される
4. 上記が Chrome・Safari・Firefox の最新版で、**ブラウザのXSLTを無効化した状態でも**同一に動作する
5. 同じZIPを再取り込みするとスキップされ、その旨が表示される
6. 取り込み後、元のダウンロードファイルが変更されていない
7. `catalog.db` を削除して再インデックスを実行すると、一覧・閲覧が完全に復元される
